# 详细实施计划书

## 1. 项目目标

开发一个桌面端自动化程序，用于彩色负片扫描图的自动去色罩、负片转正片、基础校色和批量输出。项目先完成命令行 MVP，验证核心算法，再开发图形界面和高级调色能力。

## 2. 阶段划分

项目分为 5 个阶段：

1. 立项与文档阶段
2. 核心算法 MVP 阶段
3. 批处理与诊断阶段
4. 桌面 GUI 阶段
5. 质量增强与发布阶段

## 3. 阶段 1：立项与文档

### 目标

明确项目范围、用户需求、技术路线、风险和交付标准。

### 任务

| 编号 | 任务 | 产出 |
|---|---|---|
| 1.1 | 整理胶片去色罩原理 | 原理说明 |
| 1.2 | 定义项目目标和边界 | 项目总览 |
| 1.3 | 编写需求规格 | 需求规格说明书 |
| 1.4 | 编写技术方案 | 技术方案文档 |
| 1.5 | 编写实施计划 | 详细实施计划书 |
| 1.6 | 编写测试计划 | 测试计划 |

### 完成标准

- 文档目录完整。
- 每个阶段有明确交付物。
- MVP 范围清楚，不和后续增强混淆。

## 4. 阶段 2：核心算法 MVP

### 目标

实现可运行的自动去色罩转换核心，支持单张图片处理。

### 任务

| 编号 | 任务 | 说明 |
|---|---|---|
| 2.1 | 建立 Python 项目结构 | `pyproject.toml`、`src/`、`tests/` |
| 2.2 | 实现图片读取 | 支持 JPEG、PNG、TIFF、BMP |
| 2.3 | 实现片基估计 | `auto`、`border`、`percentile`、`manual` |
| 2.4 | 实现通道归一化 | 按 RGB 通道计算 |
| 2.5 | 实现负片反相 | 输出初步正片 |
| 2.6 | 实现自动色阶 | 百分位黑白点 |
| 2.7 | 实现白平衡 | 灰世界白平衡 |
| 2.8 | 实现参数配置 | 伽马、对比度、锐化 |
| 2.9 | 写单元测试 | 覆盖核心函数 |
| 2.10 | 做合成负片测试 | 验证片基估计接近预设值 |

### 完成标准

- 单张转换命令可运行。
- 合成样例转换正确。
- 输出诊断中包含片基 RGB、黑点 RGB、白平衡增益。
- 核心函数可单独测试。

## 5. 阶段 3：批处理与诊断

### 目标

支持文件夹批处理，并提供可追踪的处理记录。

### 任务

| 编号 | 任务 | 说明 |
|---|---|---|
| 3.1 | 实现文件夹扫描 | 只处理支持格式 |
| 3.2 | 实现输出命名策略 | 默认添加 `_positive` |
| 3.3 | 实现批处理进度 | CLI 输出进度，GUI 阶段接入进度条 |
| 3.4 | 实现错误处理 | 单张失败不影响整个批次 |
| 3.5 | 输出诊断 JSON | 记录每张图参数 |
| 3.6 | 增加批处理测试 | 临时目录多图测试 |

### 完成标准

- 一个输入文件夹可以批量输出。
- 不支持格式会跳过。
- 单张失败时有错误记录。
- 所有输出图保留可追踪参数。

## 6. 阶段 4：桌面 GUI

### 目标

做一个用户可直接操作的桌面程序，提供预览、参数调节和批处理入口。

### 推荐技术

PySide6 / Qt。

### 界面模块

| 模块 | 功能 |
|---|---|
| 顶部工具栏 | 打开图片、打开文件夹、保存、批处理 |
| 左侧文件列表 | 显示待处理图片 |
| 中央预览区 | 原图/结果对比 |
| 右侧参数区 | 片基模式、伽马、对比度、白平衡、锐化 |
| 底部状态栏 | 当前状态、处理时间、错误信息 |

### 任务

| 编号 | 任务 | 说明 |
|---|---|---|
| 4.1 | 建立 GUI 项目结构 | `gui/app.py`、`gui/main_window.py` |
| 4.2 | 实现图片打开 | 单张预览 |
| 4.3 | 实现转换按钮 | 调用核心算法 |
| 4.4 | 实现参数控件 | 滑杆、下拉框、输入框 |
| 4.5 | 实现前后对比 | 分屏或切换视图 |
| 4.6 | 实现批量队列 | 文件列表和进度 |
| 4.7 | 实现后台处理 | 避免界面卡死 |
| 4.8 | 实现预设保存 | 每卷胶片保存参数 |
| 4.9 | 做 GUI 冒烟测试 | 打开、转换、保存 |

### 完成标准

- 用户不用命令行即可完成单张转换。
- 用户可以批量选择文件夹处理。
- 参数变化后可重新预览。
- 批处理时界面不冻结。

## 9. 验收标准

### MVP 验收

- 能从命令行转换单张负片。
- 能批量处理文件夹。
- 输出图像整体颜色可用，无明显蓝青色偏色。
- 支持手动片基 RGB。
- 处理过程输出诊断信息。

### 桌面版验收

- 用户能通过图形界面完成导入、预览、转换、保存。
- 用户能调整核心参数。
- 批处理有进度反馈。
- 程序异常时不崩溃，给出明确错误提示。

### 发布版验收

- Windows 用户可以无需配置 Python 运行。
- 用户手册能指导完成首次转换。
- 样片测试结果记录完整。

## 11. 关键风险

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| 裁切后无边框 | 高 | 自动色罩估计不准 | 提供高亮估计、手动 RGB、取样点 |
| 老胶片褪色严重 | 中 | 色彩难恢复 | 增加曲线和手动校色 |
| 扫描设备差异大 | 中 | 参数不通用 | 增加设备/胶片预设 |
| GUI 开发拖慢核心迭代 | 中 | 项目延期 | 先做算法和 CLI，再做 GUI |
| 用户期望接近商业软件 | 中 | 验收争议 | 明确 MVP 目标和增强路线 |

## 12. 下一步行动

1. 确认项目范围：先做 Windows 桌面程序，核心算法 Python。
2. 准备真实测试样片：至少 10 张带边框负片、10 张无边框负片。
3. 完成核心 MVP 的代码整理和测试。
4. 根据样片结果决定 GUI 前是否需要先增强算法。
5. 开始桌面 GUI 原型。
