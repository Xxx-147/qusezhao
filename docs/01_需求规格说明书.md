# 需求规格说明书

## 1. 用户画像

### 1.1 个人胶片用户

使用数码相机、手机或扫描仪 digitize 胶片，希望快速获得可看的正片。对色彩有要求，但不希望每张图都手动调曲线。

### 1.2 小型影像工作室

需要批量处理多卷胶片，关注效率、稳定性和可复现参数。需要按客户、胶片卷、日期进行输出管理。

### 1.3 技术用户

希望保留命令行和参数控制能力，能够把程序接入自己的自动化流程。

## 2. 功能需求

### 2.1 图片输入

必须支持：

- 单张图片输入
- 文件夹批量输入
- JPEG
- PNG
- TIFF
- BMP

后续支持：

- 16-bit TIFF
- 相机 RAW
- 拖拽导入
- 多文件队列

### 2.2 自动去色罩

程序必须支持以下片基估计模式：

- `auto`：自动判断是否使用边框或图像高亮区域估计。
- `border`：强制从胶片边框估计，适合 open gate 或带片边扫描。
- `percentile`：从图像高亮区域估计，适合裁掉边框的图片。
- `manual`：用户手动输入片基 RGB。

### 2.3 负片转正片

程序必须按通道完成：

- 低端密度估计
- 色罩归一化
- RGB 反相
- 黑白点拉伸
- 溢出保护

### 2.4 自动校色

第一版必须支持：

- 灰世界白平衡
- 每通道百分位色阶
- 伽马调整
- 对比度调整

后续增强：

- 中性灰取样点
- 暗部和亮部取样点
- 每通道曲线
- 每通道指数响应
- 胶片卷级别预设

### 2.5 批处理

必须支持：

- 输入文件夹
- 输出文件夹
- 保留原始文件不修改
- 自动命名输出文件
- 跳过不支持的文件格式
- 输出每张图的处理诊断数据

### 2.6 用户界面

MVP 可以先使用命令行。

桌面 GUI 阶段需要支持：

- 文件选择
- 文件夹选择
- 图片预览
- 转换前后对比
- 参数滑杆
- 自动/边框/手动片基模式切换
- 批量任务进度
- 错误提示
- 参数预设保存

## 3. 非功能需求

### 3.1 性能

- 普通 2400 万像素 JPEG 单张处理目标：5 秒内。
- 批处理时显示进度，不阻塞主界面。
- 算法内部避免不必要的全图重复拷贝。

### 3.2 质量

- 不破坏原图。
- 输出文件应可复现。
- 参数和诊断信息应可记录。
- 默认结果应该可用，允许少量后期微调。

### 3.3 易用性

- 默认模式尽量少问用户。
- 高级参数默认折叠或在 CLI 中设默认值。
- 错误信息要说明原因，例如“未检测到可用边框，已回退到高亮估计”。

### 3.4 可维护性

- 核心算法与界面分离。
- CLI、GUI 共用同一个处理核心。
- 每个处理步骤独立可测试。
- 保留算法诊断输出。

## 4. 边界和不做事项

第一阶段不做：

- 完整 RAW 解码
- AI 深度学习修复
- 灰尘划痕自动修复
- 专业级 ICC 色彩管理
- 云端同步
- 多平台安装包

第一阶段重点：

- 把核心自动化处理跑通。
- 能够稳定处理常见扫描图。
- 为后续 GUI 和高级算法留下结构。
